Selecting Your Virtual Environment as Jupyter Kernel
This is a very common step! You need to register your virt environment as a Jupyter kernel first. Here's how:

Step 1 — Install ipykernel inside your virtual environment
Make sure your virt env is activated in Git Bash, then run:

bash
pip install ipykernel
Step 2 — Register the kernel
bash
python -m ipykernel install --user --name=virt --display-name "Python (virt)"
--name=virt → internal kernel name
--display-name "Python (virt)" → what you'll see in the kernel picker
Step 3 — Select the kernel in VS Code
Since you have mongoDB_demo.ipynb open in VS Code:

Look at the top-right corner of the notebook — you'll see a button showing the current kernel (e.g., Python 3.x.x)
Click on it → a dropdown appears at the top
Select "Python (virt)" from the list
If you don't see it, click "Select Another Kernel…" → "Jupyter Kernel…" and it should appear there.

Verify it worked
Run this in a notebook cell:

python
import sys
print(sys.executable)
You should see a path pointing to your virt environment, something like:

D:/mlops/project/own/virt/Scripts/python.exe
TIP

You only need to do Steps 1 & 2 once. After that, Python (virt) will always be available as a kernel option for any notebook.
